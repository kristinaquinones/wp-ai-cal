# 2026-06-14 - Code-review remediation (v1.1.1)

## What changed

Resolved findings from a high-effort code review of the v1.1.0 refactor.

### Security / correctness
- `ajax_get_all_posts`: clamp `page`/`per_page` to positive bounds capped at
  `CALENDAR_MAX_POSTS`, closing a memory-exhaustion DoS (`-1` / huge page size).
  The sibling `ajax_get_posts` was already capped (S5).
- AI rate limiter reworked to a fixed (non-sliding) hourly window that is charged
  only after a provider call succeeds:
  - `check_ai_rate_limit()` is now a read-only gate.
  - new `record_ai_call()` increments the window with TTL anchored to the window
    start, so the hour no longer slides forward on each call.
  - all three paid handlers (`ajax_generate_outline`, `ajax_get_suggestions`,
    `ajax_check_model_health`) gate up front and record on success only, so failed
    calls no longer burn quota.
  - `ajax_check_model_health` previously made a billable call with no rate gate;
    it is now gated and recorded like the others.
- `clean_outline()` prefix-strip regex constrained to the first line (`[^\n]*?`)
  so a later heading containing a colon (e.g. `## Introduction:`) is no longer
  destroyed when output begins with a prefix phrase.

### Internal / cleanup
- Single provider roster `AIEC_Settings::get_providers()` (key => label) now drives
  `sanitize_provider()`, the settings dropdown, `get_provider_name()`, and the
  client dispatch fallback. Removed three duplicated provider lists.
- Google/Gemini now sends `generationConfig.maxOutputTokens`, so `MAX_TOKENS_CAP`
  is enforced for all four providers.
- API-key autoload hardening moved from every `admin_init` to the key's
  `add_option`/`update_option` save hooks; `ensure_api_key_not_autoloaded()` made
  public for the callback.
- Extracted shared `require_ai_access()` gate to remove copy-pasted handler preamble.

### Deliberately not changed
- The single-retry budget (`call_api_with_retry` default 1) was flagged by the
  review but is an intentional audit S4 decision (retries use blocking `sleep()`);
  CHANGELOG 1.1.0 documents it. Left as-is and added a code comment so it is not
  re-flagged.

## Validation
- `find . -name '*.php' ... -exec php -l` : clean.
- `php tests/smoke-test.php` : all checks pass. Extended with rate-limit
  (charge-on-success, read-only gate, expired-window reset), provider-roster, and
  `clean_outline` colon-heading coverage; updated the autoload test for the new
  save-hook wiring.
- `node --check` on all `assets/js/*.js` : clean.

## Follow-up / open items
- Rate limiter is still not atomic (WP transients have no atomic primitive); a
  tightly concurrent burst can over-count slightly. An object-cache
  `wp_cache_incr` approach is a possible hardening follow-up.
- Settings provider dropdown now shows plain labels (e.g. "OpenAI") instead of the
  former "OpenAI (GPT)" hints, since labels come from the shared roster.
